🔍 Phase 2: Intelligence Gathering
ENGLISH
Definition: Preliminary surveying against a target to gather maximum information.

Utility: Used during vulnerability assessment & exploitation phases.

Organizational Benefit: Helps determine their external exposure.
عربي
التعريف: مسح أولي للهدف لتجميع أقصى معلومات

الاستخدام: تُستخدم في مرحلتي تحليل الثغرات والاستغلال

الفائدة: تساعد على معرفة الانكشاف الخارجي للمؤسسة
⚠️ Phase 4: Vulnerability Analysis Most Time-Consuming
⏱️ Vulnerability Analysis is the most time-consuming phase of PTES
هذه المرحلة الأكثر استهلاكاً للوقت
ENGLISH
Transition: Assessor identifies feasible targets for further testing.

Critical Note: In phases 1-3, only info is gathered. The assessor has not touched any resources yet!
عربي
المحور: المقيّم يحدد الأهداف القابلة للاختبار

ملاحظة: في المراحل 1-3 فقط معلومات تُجمع. المقيّم لم يلمس أي موارد بعد!
💥 Phase 5: Exploitation
ENGLISH
Focus: Gaining access by bypassing security restrictions.
The Culmination: All previous work leads to the actual breach.

Methods of Access:
• Remote exploits & custom code
• Bypassing Antivirus (AV)
• Login via weak credentials
عربي
التركيز: الوصول للنظام بتجاوز القيود الأمنية
الذروة: كل العمل السابق يؤدي للاختراق الفعلي

طرق الوصول:
• استغلال عن بُعد وكود مخصص
• تجاوز برامج مكافحة الفيروسات
• تسجيل دخول بكلمات مرور ضعيفة
🔓 Phase 6: Post-Exploitation
ENGLISH
Definition: Begins after successful breach. Focuses on the value of the compromised machine.
Objective: Determine impact & what attacker could achieve once "inside".
Business Value: Identify data sensitivity & usefulness for further network compromise.
عربي
التعريف: تبدأ بعد الاختراق الناجح. تركز على قيمة الجهاز المخترق
الهدف: تحديد التأثير وما يمكن للمهاجم تحقيقه من الداخل
القيمة: تحديد حساسية البيانات وفائدتها لمزيد من الاختراق

Analysis & Pillaging
Infrastructure Analysis: Interfaces, routing, DNS, proxy servers.
Pillaging: Extracting data from programs, databases, printers.
تحليل ونهب
تحليل البنية: الواجهات، التوجيه، DNS، بروكسي
النهب: استخراج بيانات من برامج وقواعد بيانات وطابعات
🧹 Professional Cleanup Before Reporting
Cleanup is a mandatory requirement to return the system to its original state.
إلزامي — إعادة النظام لحالته الأصلية
ENGLISH
• Restoration: Reverting system settings & configs
• Removal: Deleting backdoors, scripts, artifacts
• Account Hygiene: Remove accounts created during test
عربي
• الاستعادة: إعادة إعدادات النظام
• الإزالة: حذف الـ backdoors والسكريبتات
• نظافة الحسابات: حذف الحسابات المنشأة أثناء الاختبار

⬛ Black Box Testing
ENGLISH
Knowledge: Zero knowledge provided to the tester.
Simulation: External attacker perspective.
Note: High realism, but requires extensive time for reconnaissance.
عربي
المعرفة: صفر معلومات للمختبر
المحاكاة: منظور مهاجم خارجي
ملاحظة: واقعية عالية لكن يتطلب وقتاً طويلاً للاستطلاع
⬜ White Box Testing
ENGLISH
Knowledge: Full disclosure — Source code, IP maps, Diagrams.
Simulation: Malicious insider or administrator.
Note: Maximum efficiency and technical depth.
عربي
المعرفة: كاملة — كود المصدر، خرائط IP، مخططات
المحاكاة: مدير داخلي خبيث
ملاحظة: أقصى كفاءة وعمق تقني
🔲 Grey Box Testing ⭐ Most Common Professional Model
⭐ Grey Box = the most common testing model in professional practice
الأكثر شيوعاً في الممارسة المهنية
ENGLISH
Knowledge: Partial disclosure (standard user credentials).
Why most common? Balances realism with efficiency, allowing testers to simulate real-world attacks.
عربي
المعرفة: جزئية (بيانات مستخدم عادي)
لماذا الأشيع؟ يوازن بين الواقعية والكفاءة، يسمح بمحاكاة هجمات العالم الحقيقي
🙈 Double-Blind Testing
ENGLISH
Defender is not notified — tests detection and response speed.
عربي
المدافع لا يُخطَر — يختبر سرعة الكشف والاستجابة

🌐 OSI Model — 7 Layers
Memory trick: Please Do Not Throw Sausage Pizza Away (Physical→Application)
7
Application Layer طبقة التطبيقات
Enables user interaction with network services | HTTP, FTP, IRC, SSH, DNS
6
Presentation Layer طبقة العرض
Data format conversion + encryption & compression | SSL, SSH, IMAP, FTP, MPEG, JPEG
5
Session Layer طبقة الجلسة
Manages communication sessions (open/close) | APIs, Sockets, WinSock
4
Transport Layer طبقة النقل
Reliable or unreliable delivery using transmission protocols | TCP, UDP
3
Network Layer طبقة الشبكة
Routes data between devices across multiple networks | IP, ICMP, IPSec, IGMP
2
Data Link Layer طبقة ربط البيانات
Formats data into frames + error detection | Ethernet, PPP, Switches, Bridges
1
Physical Layer الطبقة الفيزيائية
Transmits raw bits (0s and 1s) over physical medium | Cables, Fiber, Wireless, Hubs, Repeaters

🌐 Internet Protocol (IP)
ENGLISH
Operates at: Network Layer (OSI) / Internet Layer (TCP/IP)
Best-Effort Delivery: Does NOT guarantee delivery.
IPv4: 32-bit addresses
IPv6: 128-bit addresses (newer)
عربي
يعمل في: طبقة الشبكة (OSI) / طبقة الإنترنت (TCP/IP)
تسليم بأفضل جهد: لا يضمن التسليم
IPv4: عناوين 32-bit
IPv6: عناوين 128-bit (أحدث)
📦 IPv4 Header — Key Fields
ENGLISH
• Identification: Identifies fragments for reassembly
• Flags (3 bits):
  — D (Do Not Fragment): Prevents fragmentation
  — M (More Fragments): More fragments follow
• Fragment Offset: Position of fragment in original data
• TTL: Limits packet lifetime (hop count)
• Protocol: TCP=6, UDP=17
• Header Checksum: Verifies header integrity
عربي
• التعريف: يحدد الشظايا لإعادة التجميع
• العلامات (3 bits):
  — D: يمنع التجزئة
  — M: يشير لوجود شظايا أخرى
• إزاحة الشظية: موقع الشظية في البيانات الأصلية
• TTL: يحد عمر الحزمة بعدد القفزات
• البروتوكول: TCP=6, UDP=17
• Checksum: يتحقق من سلامة الرأس
🔵 TCP — Transmission Control Protocol
ENGLISH
• Connection-oriented: Session established before data transfer
• Communication closed after transfer complete
• Provides reliable data delivery
عربي
• موجه للاتصال: جلسة تُنشأ قبل نقل البيانات
• الاتصال يُغلق بعد اكتمال النقل
• يوفر تسليماً موثوقاً للبيانات

TCP Header Fields:
ENGLISH
• Source/Destination Port: Identifies applications
• Sequence/Acknowledgment Numbers: Tracks order
• Checksum: Verifies data integrity
• Window: Data allowed before ACK
• Options: Selective ACK, timestamps
عربي
• المنفذ المصدر/الوجهة: يحدد التطبيقات
• أرقام التسلسل والإقرار: يتتبع الترتيب
• Checksum: يتحقق من سلامة البيانات
• النافذة: البيانات المسموح قبل ACK
• الخيارات: إقرار انتقائي، طوابع زمنية

TCP Flags / أعلام TCP:
SYN — Synchronize
مزامنة — بدء الاتصال
ACK — Acknowledge
إقرار / تأكيد
FIN — Finish communication
إنهاء الاتصال
RST — Reset connection
إعادة تعيين الاتصال
🤝 Three-Way Handshake
Ensures both hosts are ready for communication
يضمن أن كلا الجهازين جاهزان للاتصال

StepActionDetailsعربي
① SYNHost P → Host QSYN, seq = xطلب المزامنة
② SYN-ACKHost Q → Host PACK = x+1, SYN, seq = yإقرار وطلب مزامنة
③ ACKHost P → Host QACK = y+1إقرار نهائي — يبدأ الاتصال
⚡ UDP — User Datagram Protocol

Characteristics / الخصائص

  • Connectionless — لا يُنشئ اتصال
  • No acknowledgment — لا إقرار
  • Faster — overhead أقل
  • Unreliable — غير موثوق

UDP Header (only 8 bytes!)

  • Source Port (16 bits)
  • Destination Port (16 bits)
  • Length (16 bits)
  • Checksum (16 bits) — Optional / اختياري

Use Cases / حالات الاستخدام: Video streaming, Online gaming, DNS lookups
UDP trades reliability for speed — perfect for real-time apps | UDP تبادل الموثوقية بالسرعة

📡 ICMP — Internet Control Message Protocol
ENGLISH
Error Reporting: Notifies network devices of issues — unreachable hosts, congestion, routing failures.
Operational Messages: Diagnostic & informational messages about network status.
Not for Data Exchange: Unlike TCP/UDP — only for control & management.
عربي
الإبلاغ عن الأخطاء: يُخطر الأجهزة بالمشاكل
الرسائل التشغيلية: رسائل تشخيصية عن حالة الشبكة
ليس لتبادل البيانات: فقط للتحكم والإدارة

① Ping

Check if host is reachable & measure round-trip time (RTT)
فحص إمكانية الوصول وقياس زمن الرحلة

② Traceroute

Determine the path packets take to a destination
تحديد مسار الحزم للوجهة

🔌 What is a Port? / ما هو المنفذ؟
ENGLISH
Definition: A number used to identify a specific service or application on a device within a network.
Purpose: Directs data to the right service (web server, email, file transfer).
Categories: Associated with TCP or UDP.
عربي
التعريف: رقم لتحديد خدمة أو تطبيق معين على جهاز في الشبكة
الغرض: يوجه البيانات للخدمة الصحيحة
الأنواع: مرتبطة بـ TCP أو UDP
